# A/B-тестирование: два кейса

Проект содержит два независимых A/B-теста

---

## Кейс 1. Яндекс Книги: сравнение пользователей Москвы и Санкт-Петербурга

**Бизнес-вопрос:** Пользователи из Санкт-Петербурга проводят в приложении больше времени, чем пользователи из Москвы?

**Что было сделано в SQL:**
- Подготовка данных: объединение таблиц `bookmate.audition` и `bookmate.geo`
- Фильтрация пользователей Москвы и Санкт-Петербурга
- Агрегация: сумма часов активности (`hours`) на каждого пользователя
- Выгрузка итоговой таблицы для анализа в Python

**Что было сделано в Python:**
- Проверка данных на дубликаты
- Сравнение размеров групп и статистик
- Односторонний t-тест Уэлча (α = 0.05)

**Результат:** p-value = 0.3436 → нулевая гипотеза не отвергается. Статистически значимых различий в активности между городами не обнаружено.

## Кейс 2. Интернет-магазин BitMotion Kit: тест нового интерфейса

**Бизнес-вопрос:** Упрощение интерфейса увеличивает конверсию в покупку на 3 процентных пункта в течение 7 дней после регистрации?

**Параметры теста:**
- Название: `interface_eu_test`
- Группы: A (контроль, старый интерфейс), B (тест, новый интерфейс)
- Целевая метрика: конверсия за 7 дней
- MDE: +3 п.п.
- α = 0.05

**Что было сделано:**
- Проверка корректности теста (соответствие ТЗ, равномерность групп, отсутствие пересечений)
- Очистка от пользователей, участвующих в нескольких тестах
- Фильтрация событий в пределах 7 дней после регистрации
- Расчёт необходимого размера выборки (power = 80%)
- Односторонний z-тест пропорций

**Результаты:**
| Группа | Пользователей | Покупок | Конверсия |
|--------|---------------|---------|-----------|
| A (контроль) | 4 952 | 1 377 | 27.8% |
| B (тест) | 5 011 | 1 480 | 29.5% |

- Абсолютная разница: +1.7 п.п.
- p-value = 0.0283 (статистически значимо)
- Целевой MDE (+3 п.п.) не достигнут

**Вывод:** Новый интерфейс не рекомендуется к внедрению — улучшение статистически значимо, но недостаточно для бизнес-целей.
