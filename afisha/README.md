# Анализ сервиса Яндекс Афиша

**Задача:** Выявить причины изменений в пользовательской активности осенью 2024 года и проверить гипотезы о поведении пользователей мобильных и стационарных устройств.

**Инструменты:** SQL, Python (Pandas, SciPy, Matplotlib, Seaborn), DataLens.

---

## Что было сделано

### Часть 1. SQL и дашборд
- Расчёт ключевых метрик: общая выручка, количество заказов, средний чек, DAU
- Распределение выручки по типам устройств и мероприятий
- Недельная динамика показателей
- Топ-7 регионов по выручке
- Построение дашборда в DataLens

### Часть 2. Python (EDA и проверка гипотез)
- Предобработка: конвертация валют (тенге → рубли), фильтрация выбросов по 99-му процентилю
- Создание признаков: сезон, месяц, выручка на билет
- Анализ сезонности: сравнение лета и осени
- Осенняя активность: дневная динамика, недельная цикличность
- Топ регионов и партнёров по выручке
- Проверка гипотез (t-тест Уэлча):
  - Мобильные пользователи делают больше заказов → **подтверждено**
  - Время между заказами у мобильных пользователей больше → **не подтверждено**

---

## Ключевые выводы

- Осенью заказы выросли на **17.9%** vs лето
- Мобильные устройства — основной канал (**83.7%** заказов)
- Мобильные пользователи делают в **2.6 раза** больше заказов, чем десктопные
- Топ-3 партнёра генерируют **47%** всей выручки
- Пятница — день максимальной активности
